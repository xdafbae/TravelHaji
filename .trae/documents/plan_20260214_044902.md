Untuk meningkatkan UI menjadi lebih modern dan UX yang lebih baik pada halaman `edit.blade.php` (dan juga `create.blade.php`), saya merekomendasikan beberapa peningkatan berikut:

### 1. **Pemisahan Visual Section (Card Layout)**

* Saat ini semua input "ditumpuk" dalam satu grid panjang.

* **Solusi**: Kelompokkan input ke dalam beberapa "Kartu" atau "Section" dengan judul yang jelas.

  * **Section 1: Informasi Utama** (Paket, Kota, Harga, Kapasitas)

  * **Section 2: Jadwal & Logistik** (Waktu Pergi/Pulang, Maskapai, Tour Leader)

  * **Section 3: Status** (Status Keberangkatan)

* **Benefit UX**: Memecah form panjang menjadi bagian-bagian kecil (Chunking) mengurangi beban kognitif user.

### 2. **Visual Feedback pada Pilihan Paket**

* Saat paket dipilih, harga terisi otomatis.

* **Solusi**: Tambahkan animasi atau highlight visual (misal: border flash warna hijau) pada input harga saat nilainya berubah otomatis. Ini memberitahu user "Hei, harga ini baru saja diupdate sistem lho!".

### 3. **Input Harga dengan Format Rupiah Real-time**

* Saat ini input harga hanya angka polos.

* **Solusi**: Gunakan library `AutoNumeric` atau formatter JavaScript sederhana agar saat user mengetik `1000000`, tampilannya otomatis jadi `1.000.000`.

* **Benefit UX**: Mencegah kesalahan input jumlah nol (misal: 10 juta vs 100 juta).

### 4. **Progress Stepper (Opsional tapi Bagus)**

* Jika form dirasa terlalu panjang, bisa diubah menjadi 2 langkah (Step 1: Info Paket -> Step 2: Jadwal).

* Namun untuk kasus edit, **Single Page dengan Section yang Rapi** biasanya lebih cepat (efisien) bagi admin.

### 5. **Sticky Action Bar**

* Tombol "Simpan" dan "Batal" ada di paling bawah. Jika form panjang, user harus scroll.

* **Solusi**: Buat bar tombol menjadi *sticky* di bagian bawah layar atau pojok kanan atas, sehingga selalu terlihat.

### Rencana Eksekusi:

1. **Refactor Layout**: Mengelompokkan input ke dalam `div` container dengan style Card (shadow halus, border radius).
2. **Integrasi Alpine.js Mask**: Menambahkan format Rupiah pada input harga (tampilan saja, nilai kirim tetap angka).
3. **Styling Tambahan**: Memberikan ikon pada label input untuk visual yang lebih cepat dikenali.

Apakah Anda setuju untuk menerapkan **Poin 1 (Grouping Section)** dan **Poin 3 (Format Rupiah)**? Ini adalah *low hanging fruit* dengan *impact* terbesar.
